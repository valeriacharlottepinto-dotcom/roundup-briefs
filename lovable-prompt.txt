Women, Feminism, and LGBTQIA+ news. It should have a dark, elegant aesthetic with
pride rainbow accents.

- Dark background (#0f0f1a), card backgrounds (#1a1a2e)

When "All" is active, no topic filter is applied.

TIMELINE FILTER (Row 2 ‚Äî single-select):
Clickable chip/pill buttons. Only one active at a time. Options:
  "All" (default) | "Today" | "This week" | "Last week" | "Last month" | "Last year"
Active chip gets teal coloured border and text.
Pass the selected value as ?time=today|this_week|last_week|last_month|last_year|all

SEARCH + FILTERS (Row 3):
- Search bar with magnifying glass icon ‚Äî filters by headline + summary text
  Pass as ?search=KEYWORD
- Country dropdown ‚Äî populated from /api/countries
  Pass as ?country=UK
- Source dropdown ‚Äî populated from /api/sources
  Pass as ?source=BBC+News
- Article count display on the right (e.g. "142 articles")

All filters work together ‚Äî the frontend builds a single API URL with all active
filter parameters combined.

EACH NEWS CARD should show:
- Coloured top accent bar
- Source name badge (grey)
- Category badges (teal "LGBTQIA+" or pink "Women" based on tags field)
- Topic badges (purple, from topics field ‚Äî e.g. "Reproductive Rights", "Immigration")
- Article headline (article title)
- Short summary (3 lines max, truncated)
- Date scraped + "Read ‚Üí" button linking to the original article URL

Each article object in the JSON has these fields:
  id, url_hash, title, link, summary, source, country, category, tags, topics, scraped_at

The "tags" field is a comma-separated string like "lgbtqia+, women" or "lgbtqia+".
The "topics" field is a comma-separated string like "LGBTQIA+, Human Rights" or
  "Reproductive Rights, Law & Policy". An article can have multiple topics.
The "source" field is the publication name e.g. "Gay Times", "BBC News", "Ms. Magazine".
The "country" field is the country of the publication e.g. "UK", "US", "France", "International".
